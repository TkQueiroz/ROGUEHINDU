<h1>FIGHT OF AMIR</h1>


<h1> About </h1>
My project is a 2D action survival game called Fight of Amir, built using HTML, CSS, and JavaScript with a canvas-based rendering system. The game focuses on fast-paced combat where the player must survive against increasingly difficult waves of enemies inspired by Hindu mythology.

<h1> Controls </h1>
The controls are simple and accessible:

* WASD or Arrow Keys to move
* Mouse click (or controller buttons) to attack
* E key (or equivalent on controller) to use special abilities

<h1> More about </h1>
In the game, the player can move freely around an arena and fight enemies using different weapon types such as sword, bow, mace, and spear. Each weapon has unique mechanics, including melee attacks, ranged projectiles, and piercing attacks, allowing for different playstyles. The player can level up, choose upgrades, and improve stats like damage, speed, and survivability.

The gameplay loop is based on wave progression, where enemies continuously spawn and become stronger over time. New enemy types are introduced with different behaviors such as melee, ranged, and support units. Every few waves, a boss appears with unique mechanics, making the game progressively more challenging.

I also implemented several systems to enhance the experience:

* A leveling and upgrade system with randomized power-ups
* Combo system and floating damage numbers
* Particle effects and visual feedback
* Sound effects and background music
* A global ranking system using Firebase to store high scores

For visuals, I used procedural pixel-art sprites and custom rendering instead of a game engine. This helped me better understand how graphics, animations, and the game loop work internally.

<H1> Reference </H1>

This project was inspired by games like The Binding of Isaac, especially in terms of movement and wave-based survival style, and Brotato, particularly for its fast-paced combat and upgrade system. I used these as references, but I built my own mechanics and systems from scratch and I use claude to help me because we dont have a lot of time.
